HTML

  • The head element contains information about the webpage.
  • In the head element, you first see the "!DOCTYPE html" declaration, which tells the browser what document tpe to expect when opened.
  • The "html" opening tag will signal the start, or root of our document.
  • Within the head, the 'lang="en"' attribute tells the browser the language of the page
  • The head element within the head of the html document contains metadata for the page, allowing the browser to serve the webpage to users.
  • The body element represents the visible content shown to the user.
  • Within the body you have the "header" which appears at the top of the page and contains a navigation menu, and possibly page information.
  • When using headings, they must be used from "h1" to "h6" in order to maintain readability on all devices. The biggest "h1" heading should appear only once per page.
  • In an "img" element the 'src' attribute defines the source of the image file. The 'alt' attribute describes the appearance and functionality of the image.
  • The footer element presents information about the author, copyright, contact, sitemap, and navigation.
  • HTML functions like the "frame" in the HTML, CSS, Javascript house analogy.

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • CSS Functions like the "paint" in the HTML, CSS Javascript house analogy.
  • CSS rules can be placed in the head element of your html document, but it is better to create a style sheet and link it in the head of your html document.
  • To do the above, a "link" element connects to the external file, the 'rel' attribute specifies relationship, the 'href' attribute soecifies the location of the external resource, from your current location.

Git

  • git status: checks what branch we are currently on.
  • git checkout -b branch-name: creates a new branch and switches to it.
  • git add -A: stage all changed files in the working branch.
  • git commit: commits all staged changes to your repository.
  • A "-m" flag can be used directly after commit in the git commit command, to leave a short descriptive message of the changes implemented.
